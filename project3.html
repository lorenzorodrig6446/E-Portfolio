Course AERSP 304 — System Dynamics & Controls
Timeline Spring 2025
Type Group Project

Background

From Equations on Paper
to Hardware in Hand

Control theory is a cornerstone of aerospace engineering — but its real power only becomes apparent when applied to a physical system. The Funduino project was designed to bridge exactly that gap, translating the mathematical language of transfer functions and feedback loops into a working, hardware-in-the-loop motor control system.

The actuator at the center of the project was a 12V DC gear motor equipped with a rotary encoder, driven by an Arduino Mega microcontroller through an L298N H-bridge motor driver. The task: derive the motor's theoretical transfer function from first principles, experimentally identify it from real data, and ultimately design a feedback controller capable of driving the motor to a precise target speed — entirely autonomously.

The project unfolded across two phases. In Part I, the governing electromechanical equations were derived using Kirchhoff's voltage law and Newton's second law in rotational form, producing theoretical transfer functions in the Laplace domain. These were validated against step-response data logged via MATLAB serial communication at six input voltages. In Part II, three progressively sophisticated control strategies — open-loop, closed-loop, and PID — were implemented both in simulation and on the physical hardware, with real-time performance compared against theoretical predictions.

This project is a direct demonstration of how systems dynamics and controls principles apply far beyond the classroom. The same analytical methods underpin attitude control systems, engine governors, and autonomous vehicle platforms across the aerospace and defense industries.

Project Pipeline

📐
Theoretical Derivation
Derived velocity and position transfer functions G(s) and H(s) from DC motor governing equations using Kirchhoff's voltage law and Newton's rotational second law. Applied Laplace transforms with zero initial conditions to obtain frequency-domain models.
↓
🔬
Experimental Identification
Applied step voltage inputs from 4V to 9.6V via Arduino and logged RPM encoder data in MATLAB. Extracted DC gain K and time constant τ for each input. Averaged across six runs to yield G(s) = 46.61 / (0.114s + 1).
↓
📈
Chirp Signal Validation
Tested the estimated model against a sinusoidal chirp input of increasing frequency, comparing experimental encoder data with MATLAB-simulated predictions. Confirmed first-order model adequacy and characterized frequency-domain error behavior.
↓
⚙️
Controller Design & Implementation
Designed open-loop, closed-loop (unity feedback), and PID controllers in MATLAB simulation, then implemented each on the physical Arduino hardware. Root locus analysis established stability margins; PID gain K_i = 2.051 was tuned for fastest stable convergence.

System Modeling

Verified the model structure against experimental step-response data across multiple input voltages and computed standard deviations for all identified parameters.

💻

Arduino & MATLAB Programming

Wrote and debugged Arduino firmware for real-time RPM acquisition using encoder interrupts and PWM voltage command delivery. Developed MATLAB scripts for serial data logging, transfer function simulation using lsim, chirp signal generation, and comparative error analysis across all tested voltages and control architectures.

🎛️

PID Controller Tuning

Designed and experimentally tuned a PID controller targeting 80% of the motor's maximum RPM. Used MATLAB's PID tuner for initial gain estimation, then iteratively refined on physical hardware. The final controller (K_i = 2.051, K_p = K_d = 0) achieved stable convergence within one second — validated against the simulated step response.

Hardware & Results

Complete experimental hardware setup for the Funduino project. The Elegoo Mega 2560 Arduino (center-left) interfaces with the 12V DC gear motor through an L298N H-bridge driver (top center), with rotary encoder feedback wired directly back to the microcontroller interrupt pins. A dedicated power bank supplies the motor rail independently from the 5V logic supply, preventing motor transients from disrupting the Arduino.

Averaged Transfer Function Parameters

Averaged DC gain K and time constant τ extracted from step-response experiments across six input voltages (4V to 9.6V), along with their standard deviations. These values parameterize the estimated first-order transfer function that underpins all subsequent controller design and simulation work.

Key Analytical Results

Estimated Transfer Function

G(s) = 46.61 / (0.114s + 1)

First-order model averaging K and τ across all voltage runs. The motor's electrical dynamics are far faster than its mechanical dynamics, producing a single-pole, overdamped response well-captured by this simplified form.

Final PID Controller

Gf(s) = 2.051 / (0.114s² + s + 2.051)

Closed-loop PID with Kp = Kd = 0, Ki = 2.051. Achieved stable convergence to target RPM within ~1 second on physical hardware — significantly outperforming open-loop and simple closed-loop alternatives.

Core mathematical results of the project. The estimated transfer function drove all simulation and design decisions; the PID controller transfer function represents the experimentally validated final implementation on the Arduino hardware.

Summary

Controls Theory,
Applied

The Funduino project closed the loop between theory and practice. Each phase — derivation, identification, validation, and control — built directly on the last, resulting in a working hardware controller that matched simulation predictions with strong fidelity. The PID controller outperformed both open- and closed-loop alternatives, converging to the target speed within one second on the physical motor.

Closed-Loop Performance — PID vs. Simulation Sim Rise Time: 0.125 s  |  Sim Settling Time: 0.223 s
Target RPM: 370  |  PID Convergence: < 1 second

Hardware response consistent with simulation — PID outperformed open-loop and closed-loop alternatives in both accuracy and convergence speed.

  • Theory meets hardware: Deriving the transfer function from first principles and then validating it experimentally reinforced that mathematical models are powerful — but real systems always introduce friction, noise, and nonlinearity that idealized models must account for.
  • First-order approximations are a powerful engineering tool: The DC motor's electrical dynamics are far faster than its mechanical dynamics, making a simplified first-order model surprisingly accurate. Knowing which physics dominates is a core systems engineering skill.
  • PID outperforms simpler controllers: Open-loop and closed-loop systems both completed the basic task, but the PID controller's iterative error minimization produced dramatically faster, more accurate convergence — at the cost of tuning effort and implementation complexity.
  • Data acquisition is as critical as control design: Reliable RPM measurement required careful interrupt handling and sampling interval selection. Poor sensing translates directly to poor control performance, regardless of algorithm sophistication.
  • Embedded systems work is real engineering: Wiring the H-bridge, managing separate power rails, debugging serial communication, and calibrating encoder counts were unglamorous but essential — a preview of the hardware integration challenges common in aerospace and defense development programs.
